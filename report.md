# 농산물 가격 예측 — ARIMA / SARIMA 모델링 과정

## 1. 프로젝트 개요

네팔 카트만두 칼리마티 시장의 농산물 가격 데이터를 활용해 시계열 모델(ARIMA, SARIMA)로 가격을 예측한다.

---

## 3. 문제 발견 — 잘못된 계절 주기

초기 코드의 SARIMA 설정:

```python
model = SARIMAX(
    train,
    order=(1, 1, 1),
    seasonal_order=(1, 1, 1, 12)  # m=12
)
```

데이터는 **일별(daily)** 관측값인데 `m=12`로 설정되어 있었다.  
SARIMA에서 `m`은 "몇 개의 관측값이 1주기인가"를 의미하므로, 일별 데이터에서 `m=12`는 **12일 주기**로 계절성을 학습하게 된다.

농산물 가격의 실제 계절성은 **1년(연간) 주기**이므로 일별 데이터라면 `m=365`가 맞다.  
그러나 `m=365`는 상태공간 행렬 크기가 기하급수적으로 커져 **사실상 계산 불가**하다.

### 날짜 간격 분포 확인

```
1일 간격: 2,364건
2일 간격: 102건
3일 이상: 35건 (최대 46일 공백 존재)
```

날짜가 불규칙하게 누락되어 있어 전처리도 필요한 상태였다.

### 이 오류가 위험한 이유

**① 에러가 발생하지 않는다 — 침묵하는 오류**

`m=12`로 설정해도 statsmodels는 아무 경고 없이 정상 실행된다. 출력된 MAE/RMSE 수치도 그럴듯한 범위에 있어 결과만 봐서는 문제를 인식하기 어렵다. 잘못된 설정임에도 코드가 "작동한다"는 착각을 준다.

**② 틀린 결론을 유도한다 — 모델 선택 오류**

기존 결과:

| 모델 | MAE |
|------|-----|
| ARIMA | 19.34 |
| SARIMA | 23.56 |

SARIMA가 ARIMA보다 성능이 나쁘게 나왔다. 이 결과만 보면 "계절성을 추가해도 효과가 없다" 또는 "ARIMA로 충분하다"는 잘못된 결론을 내릴 수 있다. 실제로는 SARIMA가 더 적합한 모델인데, 잘못된 주기 설정 때문에 오히려 나쁜 모델처럼 보인 것이다.

**③ 학습한 패턴 자체가 틀렸다**

`m=12`는 "12개의 관측값마다 계절이 반복된다"는 뜻이다. 일별 데이터에서는 **12일 주기**로 계절성을 학습하게 된다. 농산물 가격과 아무 관련 없는 패턴을 계절성으로 간주하고 학습한 셈이므로, 예측 자체의 신뢰도가 낮아진다.

**④ 수정 후 결과와 비교**

| 모델 | 기존 MAE | 수정 후 MAE | 차이 |
|------|----------|-------------|------|
| ARIMA | 19.34 | 12.54 | -6.80 |
| SARIMA | 23.56 | 10.33 | **-13.23** |

SARIMA는 수정 후 MAE가 절반 이상 줄었다. 기존 설정이 얼마나 잘못된 방향으로 학습하고 있었는지를 보여준다.

## 4. 해결 방법 — 월별 리샘플링

일별 데이터를 **월별 평균**으로 집계하고, 빠진 달은 선형 보간으로 채운다.  
월별 데이터에서는 `m=12`가 정확히 **12개월 = 1년 주기**를 의미하므로 기존 파라미터를 그대로 사용할 수 있다.

```python
price = data["Average"].resample("MS").mean().interpolate("linear")
# 결과: 96개월 (2013-06 ~ 2021-05)
```

- 일별 노이즈 제거
- 날짜 불규칙 문제 해소
- SARIMA 계절 주기 정상화 (m=12 = 연간 주기)

### 4-1. 리샘플링 집계 기준

월별로 묶을 때 어떤 값을 대표값으로 쓸지 선택해야 한다.

| 집계 방법 | 설명 | 적합한 경우 |
|-----------|------|-------------|
| **평균 (mean)** | 해당 월 전체 거래일의 평균 | 가격처럼 연속적으로 변동하는 값 — 이상치 영향을 분산시킴 **(본 프로젝트 사용)** |
| 중앙값 (median) | 월 중간값 | 이상치가 많을 때 더 안정적 |
| 마지막값 (last) | 월말 기준 가격 | 주가처럼 종가가 의미 있는 경우 |
| 합계 (sum) | 월 전체 합산 | 거래량, 강수량 같은 누적 지표 |

가격 데이터는 **평균**이 가장 자연스럽다. 한 달 안에서도 가격이 오르내리므로 평균이 그 달의 대표 가격 수준을 잘 나타낸다.

### 4-2. 결측 달 처리 기준

날짜 공백(최대 46일)으로 인해 특정 달에 거래 기록이 아예 없을 수 있다. 이 경우 `resample("MS").mean()`은 `NaN`을 반환한다.

| 보간 방법 | 설명 | 적합한 경우 |
|-----------|------|-------------|
| **선형 보간 (linear)** | 앞뒤 값을 직선으로 연결 | 결측이 1~2개월로 짧을 때 **(본 프로젝트 사용)** |
| 전방 채우기 (ffill) | 직전 달 값을 그대로 복사 | 값이 급변하지 않는 경우 |
| 후방 채우기 (bfill) | 다음 달 값을 그대로 복사 | 드물게 사용 |
| 계절 평균 대체 | 같은 달의 다년도 평균으로 채움 | 결측이 많고 계절성이 강할 때 |

본 데이터에서 결측 달은 소수이고 가격이 점진적으로 변하는 경향이 있으므로 **선형 보간**이 적절하다.

### 4-3. 리샘플링 시 주의할 점

- **`resample("MS")`** 는 각 월의 첫날(`Month Start`)을 인덱스로 설정한다. `"M"`(월말 기준)과 혼동하지 않도록 주의.
- 집계 후 데이터 수가 줄어드므로 **train/test 분할 시 절대 수량을 확인**해야 한다. 본 프로젝트는 96개월 중 80%(약 77개월) 학습, 20%(약 19개월) 테스트.
- 월별 집계는 **정보 손실**을 수반한다. 일중 가격 변동, 주간 패턴 등은 사라진다. 단기 예측보다 **중장기 추세·계절성 분석**에 적합하다.

---

## 5. 모델

### ARIMA(1, 1, 1)

- 차분(d=1)으로 정상성 확보
- AR(1): 이전 시점 가격 반영
- MA(1): 이전 오차 반영
- 계절성 미반영

```python
model = ARIMA(train, order=(1, 1, 1))
```

### SARIMA(1, 1, 1)(1, 1, 1, 12)

- ARIMA에 계절 성분 추가
- 계절 차분(D=1), 계절 AR/MA(1) 적용
- m=12: 12개월(1년) 주기 계절성 학습

```python
model = SARIMAX(
    train,
    order=(1, 1, 1),
    seasonal_order=(1, 1, 1, 12)
)
```

### 공통 설정

- Train/Test 분할: 80% / 20%
- 평가 지표: MAE, RMSE

---

## 6. 결과

| 모델 | 데이터 | MAE | RMSE |
|------|--------|-----|------|
| ARIMA | 일별 (기존) | 19.34 | 22.48 |
| SARIMA | 일별, m=12 오류 (기존) | 23.56 | 27.35 |
| **ARIMA** | **월별 리샘플링** | **12.54** | **16.79** |
| **SARIMA** | **월별 리샘플링, m=12 수정** | **10.33** | **13.33** |

### 분석

1. **월별 리샘플링만으로** ARIMA 성능이 크게 향상됐다 (MAE 19.34 → 12.54).  
   일별 노이즈 제거 효과로, 모델이 실질적인 가격 흐름에 집중하게 됐다.

2. **SARIMA가 ARIMA를 역전했다** (MAE 10.33 vs 12.54).  
   기존에는 계절 주기 오류로 SARIMA가 더 나쁜 결과를 냈지만, 수정 후 연간 계절성을 올바르게 학습하면서 성능이 향상됐다.

3. **계절성 반영이 핵심이다.**  
   토마토 가격은 몬순, 수확철 등 연간 패턴이 뚜렷하기 때문에 계절 성분을 가진 SARIMA가 더 적합한 모델이다.

---

## 7. 결론

- 일별 데이터에 `m=12`를 그대로 쓰는 것은 잘못된 계절 주기 설정이다.
- 월별 리샘플링 후 `m=12`를 적용하는 것이 연간 계절성을 올바르게 반영하는 방법이다.
- 최종적으로 **SARIMA(1,1,1)(1,1,1,12) on 월별 데이터**가 가장 좋은 성능을 보였다 (MAE: 10.33, RMSE: 13.33).
