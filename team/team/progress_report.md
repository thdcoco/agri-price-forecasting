# 전력수요 SARIMA vs SARIMAX 분석 — 진척 보고서

작성일: 2026-06-02

---

## 1. 프로젝트 목표

한국 전국 전력수요 데이터를 SARIMA(외생변수 없음)와 SARIMAX(기온 외생변수)로 분석·비교하고, **2026년 상반기 전력수요를 예측**한다.

핵심 메시지: *"전력수요는 기온에 강하게 반응하며, 기온을 외생변수로 넣은 SARIMAX가 SARIMA보다 변동을 잘 설명한다."*

---

## 2. 사용 데이터

### 전력수요
- 출처: 한국전력거래소 시간별 전국 전력수요량
- 기간: 2013 ~ 2025 (연도별 CSV 6개 병합)
- 처리: 24개 시간값 일평균 → 일별 수요(`load_mw`) → **GW 단위 변환** (÷1000, 수렴 안정성)
- 중복 날짜 `drop_duplicates('날짜')` 처리

### 기온 (외생변수)
- 출처: 기상청 전국 일별 기온
- 기간 1: 2013-01-01 ~ 2025-12-31 (`ta_20260602175659.csv`)
- 기간 2: 2026-01-01 ~ 2026-06-01, 152일 (`ta_20260602175713.csv`)
- 컬럼: `날짜, 지점, 평균기온, 최저기온, 최고기온`
- 인코딩: cp949, 헤더 6줄 skip
- **전국 데이터 사용** (초기에 서울 지점 108 필터 사용했다가 전국으로 교체)

## 3. 외생변수 설계 (feature_design.md 기반)

| 모델 | 변수 | 수식 |
|---|---|---|
| **M1** 평균기온 기반 | CDD_avg, HDD_avg | `max(평균기온-24,0)`, `max(18-평균기온,0)` |
| **M2** 최고·최저 분리 | CDD_max, HDD_min | `max(최고기온-24,0)`, `max(18-최저기온,0)` |
| **M3** 열대야 추가 | CDD_max, HDD_min, TROP | M2 + `max(최저기온-23,0)` |

**설계 원칙**: 일교차(최고-최저)는 변수로 사용 불가 — 여름 일교차 15℃와 겨울 일교차 15℃가 수요에 정반대 영향을 미치므로 해석이 깨진다.

## 4. 모델링 방법

### SARIMA 차수
- `order = (p, 1, q)` — d=1 (1차 차분)
- `seasonal_order = (P, 1, Q, 7)` — D=1, s=7 (주간 계절성)
- 그리드 서치: p,q ∈ {0,1,2}, P,Q ∈ {0,1} → **36개 조합**
- 선택 기준: **AIC 최소화**
- 동일 최적 차수를 SARIMA·M1·M2·M3 전부 적용 (공정 비교)

### 검증 구간
- 학습: 2013-01-01 ~ 2022-12-31
- 검증: **2023-01-01 ~ 2024-12-31** (4계절 × 2년 — 여름·겨울 피크와 봄·가을 저부하 구간 모두 포함)

### 예측 구간
- 전체 학습(2013~2025) 후 **2026-01-01 ~ 2026-06-01** 예측
- SARIMAX: 실제 2026 기온으로 만든 exog 입력
- SARIMA: 외생변수 없이 과거 패턴만 연장

## 5. 주요 결과

### 성능 비교 (검증: 2024년 여름)

| 모델 | RMSE (GW) | MAPE | 비고 |
|---|---|---|---|
| SARIMA | 14.308 | 18.02% | 기온 무시 |
| SARIMAX-M1 | **12.111** | **15.53%** | **최우수** |
| SARIMAX-M2 | 14.020 | 17.67% | |
| SARIMAX-M3 | 13.443 | 17.04% | |

**주목할 점**: M2(물리적 근거 반영)가 M1(단순 평균기온)보다 성능이 낮음 → 여름 검증셋에서 평균기온이 냉방 효과를 더 잘 포착하는 것으로 해석. AIC 기준도 M1이 최선.

### 외생변수 계수 (모두 p<0.001 유의)

| 모델 | 변수 | 계수 |
|---|---|---|
| M1 | CDD_avg | +0.856 GW/도일 |
| M1 | HDD_avg | +0.247 GW/도일 |
| M2 | CDD_max | +0.083 GW/도일 |
| M2 | HDD_min | +0.046 GW/도일 |
| M3 | CDD_max | +0.081 GW/도일 |
| M3 | HDD_min | +0.047 GW/도일 |
| M3 | TROP | +0.721 GW/도일 |

M3에서 열대야(TROP) 계수가 압도적으로 큼 — M2에서 분리되지 않던 야간 냉방 효과를 TROP가 흡수.

### 2026 상반기 예측
- SARIMA: 과거 주간 패턴 연장, 비교적 평탄
- SARIMAX-M1: 2026년 5월 이후 기온 급상승 구간에서 예측값도 상승 → **기온 굴곡 추종 확인**
- 최종 채택 모델: **SARIMAX-M1** (AIC 최소, RMSE 최소, MAPE 최소)

## 7. 주요 기술 이슈 및 해결

### acorr_ljungbox API 변경
```python
# 구버전 (tuple 반환)
_, lb_pval = acorr_ljungbox(resid, lags)

# 신버전 (DataFrame 반환) — 수정 적용
lb_df = acorr_ljungbox(resid, lags=np.arange(1, 11, 1))
lb_pval = lb_df['lb_pvalue'].values
```

### 기온 데이터 전국 교체
- 초기: 서울 지점(108) 필터 → `df[df['지점'] == 108]`
- 변경: 전국 데이터 파일로 교체, 필터 제거 → `.dropna(subset=['평균기온'])` 사용
- 이유: 전국 수요에 서울 기온만 쓰는 것이 대표성 부족

### GW 단위 변환
- MW 단위(수만 단위)로 SARIMAX 돌리면 최적화 수렴 실패
- 해결: ÷1000 → GW 단위로 변환 후 모델링

### 그리드 서치
- 36개 SARIMA 조합을 전부 AIC 계산 → 최적 차수 1개 선택
- 4개 모델(SARIMA/M1/M2/M3) 동일 차수 적용 → 공정 비교
