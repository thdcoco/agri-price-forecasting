# 외생변수 설계: 평균기온 vs 최고·최저 분리

## 결론 먼저
- **일교차(최고 − 최저)를 변수로 쓰지 말 것.** 겨울 일교차 15℃(추움)와 여름 일교차 15℃(더움)는
  수요에 정반대 영향 → 같은 숫자가 정반대 의미가 되어 해석이 깨진다.
- 최고와 최저는 **빼서 비교하는 대상이 아니라, 서로 다른 수요를 일으키는 별개의 원인**으로 본다.
  - 여름 냉방 = 낮 더위 → **최고기온**
  - 겨울 난방 = 새벽 추위 → **최저기온**

---

## 외생변수 설계안

### 모델 1 — 평균기온 기반 (베이스라인)
```
CDD_avg = max(평균기온 - 24, 0)
HDD_avg = max(18 - 평균기온, 0)
```
- 장점: 단순
- 한계: 열대야(최저도 높은 날) 같은 상황을 구분 못 함

### 모델 2 — 최고·최저 분리 (물리적 근거 반영)
```
CDD_max = max(최고기온 - 24, 0)   # 여름에만 값 발생 (낮 폭염 → 냉방 피크)
HDD_min = max(18 - 최저기온, 0)   # 겨울에만 값 발생 (새벽 한파 → 난방)
```
- 두 변수가 계절적으로 거의 겹치지 않음 (여름엔 HDD=0, 겨울엔 CDD=0)
  → "여름/겨울 구분이 애매하다"는 문제가 변수 설계로 자동 해소됨
- 각 변수가 자기 계절을 전담하므로 서로 충돌하지 않음

### 모델 3 — 열대야 추가 (선택, 심화)
```
CDD_max  = max(최고기온 - 24, 0)   # 낮 더위
TROP     = max(최저기온 - 23, 0)   # 밤 더위(열대야) → 야간 냉방 미차단
HDD_min  = max(18 - 최저기온, 0)   # 겨울 난방
```
- "낮 더위 + 밤 더위가 둘 다 냉방을 끌어올린다"까지 설명
- 주의: 변수 증가 → 과적합 위험. **넣었을 때 AIC가 실제로 내려가는지 확인** 필요

---

## "비교"의 의미 — 변수끼리가 아니라 모델 버전끼리

| 비교 | 내용 |
|------|------|
| 모델 1 vs 모델 2 | 평균 1개 vs 최고·최저 분리 → 오차(RMSE/MAPE)·AIC 개선 여부 |
| 모델 2 vs 모델 3 | 열대야 변수 추가가 실제로 설명력을 높이는지 |

- 핵심 스토리: 단순히 변수를 늘린 게 아니라
  **물리적 근거(낮 더위=최고 / 새벽 추위=최저)로 설계했다**는 점.
- 발표에서 "왜 굳이 최고·최저를 따로 봤나?"에 대한 답이 됨.

---

## 분석 절차 (각 모델 동일하게)
1. ADF 정상성 검정 → d 결정
2. ACF/PACF → (p, q) 추정, 계절성 s=7
3. SARIMAX 적합 (exog만 모델별로 교체)
4. 비교표: AIC / RMSE / MAPE / 외생변수 계수·p-value
5. AIC가 내려가고 계수가 유의할 때만 변수 추가를 채택

---

## 주의점
- 기준온도(24℃ / 18℃ / 23℃)는 국내 실측에 맞게 조정 가능 — 민감도 확인 권장
- 변수 추가 시 다중공선성 점검 (최고·최저·평균을 동시에 넣지 말 것)
- 외생변수가 많아질수록 미래 예측 시 모든 외생변수의 미래값이 필요함
