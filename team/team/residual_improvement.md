## FAIL의 원인

### 1. 대용량 데이터 효과 (주요 원인)
학습 데이터가 4,017일로 매우 크다. 표본이 클수록 잔차에 남아있는 극히 미세한 자기상관도 통계적으로 유의하게 검출된다. 잔차 ACF 그래프(fig6)를 보면 lag 0을 제외한 모든 시차가 신뢰구간 내에 수렴하므로 **실질적 자기상관 문제는 없는 수준**이다.

### 2. 모델이 설명하지 못한 구조적 패턴
- **명절 효과**: 설·추석 연휴 수요 급감이 잔차에 반복 패턴으로 남음
- **코로나 기간(2020~2021)**: 산업·상업 수요 이상으로 일시적 구조 변화
- **장기 수요 트렌드**: 전기차 보급, 산업구조 변화 등으로 기온 외 요인이 누적

## 개선 방안

### 단기 (수업 범위 내)

**① 명절 더미 변수 추가**
- 설·추석 연휴 날짜를 0/1 더미로 생성해 SARIMAX exog에 추가
- 명절 직전 1~2일, 연휴 기간, 연휴 직후 1~2일을 각각 별도 더미로 설계
- 예상 효과: 연휴 구간 잔차 스파이크 감소

```python
# 구현 예시
holidays = pd.to_datetime([
    '2013-02-09', '2013-02-10', '2013-02-11',  # 설
    # ... 2013~2025 설·추석 날짜 목록
])
df['holiday'] = df.index.isin(holidays).astype(int)
```

**② 차수 상향 검토**
- 현재 (2,1,2)(1,1,1,7)에서 p 또는 q를 3으로 높여 AIC 비교
- 단, 과적합 위험이 있으므로 검증 RMSE가 개선되는 경우에만 채택

### 장기 (심화 분석)

**③ 연간 계절성 추가**
- 현재 s=7(주간)만 반영, 연간 계절성(s=365)은 미반영
- 일별 데이터에 s=365 적용 시 계산량이 매우 커서 현실적으로 Prophet, STL+ARIMA 등 대안 모델 검토 필요

**④ ARIMA-GARCH 결합**
- 잔차 분산이 일정하지 않은 이분산성(heteroskedasticity)이 관찰됨
- ARIMA로 평균 구조, GARCH로 분산 구조를 따로 모델링하면 잔차 독립성 개선 가능
- 수업 범위 밖이므로 향후 과제로 분류
